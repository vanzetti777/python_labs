**Класс** - Product

**Защищённые атрибуты экземпляра** - имя, цена, состояние и кол-во на складе.
## Управление доступом через @property

Состояние экземпляра контролируем через @property:
- **Геттер (@property)** - контролируем чтение
- **Сеттер (@price.setter)** - контролируем изменение с проверками

Геттеры я сделала для всех атрибутов, чтобы можно было получить их значения. Сеттер сделала только для цены, потому что цена может меняться.

## Атрибуты класса

**Атрибуты класса** - скидка и ндс

## Валидация (файл validate.py)

В файле Validate я вынесла все проверки через @classmetod, такие как:
- **Проверка типа данных** 
- **Проверка диапазона** 
- **Проверка допустимых значений** (статус должен быть из списка)

## Магические методы

**Магические методы:**
- **`__str__`** - красивый вывод для пользователя

- **`__repr__`** - вывод для разработчика

- **`__eq__`** - сравнение товаров

## Бизнес-методы

**Бизнес методы:**

### 9. Ошибки валидации при создании объекта (через `Validator`)

| Проверка | Попытка | Ошибка |
|----------|---------|--------|
| **Тип названия** | передать число вместо строки | `название должно быть строкой` |
| **Пустое название** | передать пустую строку | `название не может быть пустым` |
| **Длина названия** | передать > 20 символов | `название не может быть длиннее 20 символов` |
| **Отрицательная цена** | передать -1000 | `цена не может быть отрицательной` |
| **Цена слишком большая** | передать 1_500_000 | `цена не может быть больше 1000000` |
| **Тип цены** | передать строку | `цена должна быть числом` |
| **Отрицательное количество** | передать -5 | `количество не может быть отрицательным` |
| **Тип количества** | передать 5.5 (не целое) | `количество должно быть целым числом` |
| **Некорректный статус** | передать "available" | `статус должен быть одним из: ['in_stock', 'out_of_stock', 'discontinued', 'preorder']` |
| **Тип статуса** | передать число | `статус должен быть строкой` |
